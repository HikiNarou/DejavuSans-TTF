Penjelasan Diagram Arsitektur Berlapis:
1.	Presentation Layer:
	o	Berisi komponen antarmuka pengguna (UI)
	o	Activities dan Fragments untuk tampilan utama
	o	ViewModels untuk logika presentasi dan state management
	o	Adapters untuk menampilkan data dalam RecyclerViews
	o	Komponen navigasi, UI, animasi, dan sistem theming
2.	Domain Layer:
	o	Layer bisnis yang independen dari UI dan data
	o	Use Cases melaksanakan fitur-fitur spesifik aplikasi
	o	Entities mewakili model domain inti
	o	Repository Interfaces mendefinisikan kontrak untuk akses data
	o	Domain Services untuk logika bisnis kompleks
3.	Data Layer:
	o	Mengelola akses dan manipulasi data
	o	Repository Implementations menyediakan data ke Domain layer
	o	Remote Data Source mengelola koneksi API
	o	Local Data Source menangani database lokal
	o	Data Mappers untuk mengkonversi antara model data dan domain
4.	Infrastructure Layer:
	o	Menangani fungsionalitas teknikal dan integrasi dengan sistem eksternal
	o	Networking untuk komunikasi HTTP
	o	Framework database seperti Room
	o	Plugin Framework untuk ekstensi sumber konten
	o	Integrasi dengan layanan cloud dan sistem
